How I Built the Dashboard

The actual prompts I used — no coding required

1

The Big Ask

"I want to build a Centennial Conference Baseball Scouting Dashboard as a React web app. This is for a D3 college baseball player at Muhlenberg College. His 2026 season starts February 21.

The app lets the user select an upcoming opponent from the Centennial Conference and generates a full scouting report with stats and AI analysis.

Scrape the stats from the conference website. Store it as JSON. Build a dashboard with head-to-head comparisons, charts, player tables, and an AI scouting report button..."
From this one message, Claude built the entire scraper (to grab stats from 10 team websites) and the full dashboard structure. I just described what I wanted it to do.
2

Testing & Giving Feedback

"Some of the top hitters have like a really small sample size of at bats. And so I think there should be like a cut off for at bats so that there's a significant enough sample size for them to be considered a top hitter.

Also if I sort by stolen bases, it's only going to sort out of the 5 that are originally there. It should pull from ALL qualified players on the team."
I noticed a problem while testing and just described what was wrong. I didn't know how to fix it — I just said what I wanted to be different. Claude handled the logic.
3

Visual Feedback

"For the per game stats it should be more obvious which team has the edge. I don't like the checkmark either. Remember this is for Muhlenberg."
I just said what I didn't like. No need to know CSS or design — just "I don't like the checkmark" and "make it more obvious." Claude redesigned it.
4

When Things Break

"I tried to generate a scouting report and got this error: [pasted error message]"
Things broke constantly. I didn't need to understand why — I just pasted the error and Claude figured out what went wrong and fixed it.
5

Asking for Help

"Can you write explicit instructions? I'm not that technical."

"Wait please include how you open the project folder."

"Wait how do I edit the .env file?"
When I didn't understand something, I just asked for simpler instructions. You don't need to pretend you know things — just ask.
6

The Meta Moment

"Any technical jargon needs to be dumbed down as much as possible because my cousin is a freshman in college. He's not the most cerebral guy. He doesn't have a lot of programming background at all — none.

I don't have the programming background myself, so I want to highlight kind of the process..."
Claude adapts to any audience. I told it who you are and it adjusted how it explained things. That's why the guide is written the way it is.

The Point

I built this entire dashboard — scraping stats from 10 teams, charts, tables, AI scouting reports, print functionality — and I don't know how any of the code actually works. I just kept describing what I wanted until it existed.
